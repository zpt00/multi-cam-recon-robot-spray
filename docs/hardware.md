# 四相机采集平台硬件设计文档

## 1. 四相机平台设计目标

### 1.1 需求背景

本硬件平台专为工业机器人喷涂轨迹规划中的工件三维重建任务而设计。在实际生产环境中，待喷涂工件（如汽车零部件、五金件、异形管材等）通常具有复杂的三维几何结构，单视角深度相机无法一次捕获全部可见表面。因此，需要部署多台深度相机从不同方位同步采集，以获取工件完整的表面三维信息。

### 1.2 设计目标

| 设计维度 | 目标指标 | 说明 |
|---------|---------|------|
| 覆盖范围 | 工件表面覆盖率 > 95% | 四相机环绕布局，消除视觉盲区 |
| 空间分辨率 | 深度图 1280×720 @ 15 FPS | D435 在该分辨率下的推荐工作距离 0.3~1.5m |
| 外参稳定性 | 平移漂移 < 2mm，旋转漂移 < 0.3° | 工业现场振动环境下 8 小时连续运行 |
| 标定精度 | 平移误差 < 5mm，旋转误差 < 1° | 满足后续 ICP 精配准的初始条件 |
| 安装方式 | 固定式，Eye-to-Hand 布局 | 相机不随机器人运动，独立观察工作空间 |
| 电气兼容 | USB 3.0 四通道同步采集 | 单台 PC 控制，无需额外同步硬件 |

### 1.3 传感器选型：Intel RealSense D435

选用 Intel RealSense D435 深度相机，原因如下：

- **主动立体红外 (Active Stereo IR)**：D435 采用双目红外相机 + 红外点阵投射器方案。投射器向场景发射伪随机点阵图案，增强低纹理表面（如纯色金属工件、塑料件）的纹理特征，显著提升立体匹配的可靠性和深度精度。
- **全局快门 (Global Shutter)**：红外双目相机使用全局快门传感器，在工业现场存在机械振动时不会产生卷帘快门常见的拖影失真。
- **视场角**：深度 FOV 为 87°×58°（H×V），彩色 FOV 为 69°×42°。四台相机按 ~60°~90° 间隔布置时，相邻相机视场重叠区域 > 30%，满足 ICP 配准对特征重叠的要求。
- **USB 3.0 接口**：单相机最大数据吞吐量约 200 MB/s，四台相机总计约 800 MB/s，对 USB 控制器数量和主板带宽有明确约束（详见第 6 节）。
- **SDK 成熟度**：Intel 提供跨平台 `librealsense2` SDK，Python 绑定 (`pyrealsense2`) 与 Open3D / OpenCV 无缝集成，支持多相机同步配置（全局时间戳、硬件触发）。

可选 D435i 型号内置 BMI055 六轴 IMU（加速度计 + 陀螺仪），可用于振动监测或辅助视觉惯性里程计，但在本系统的固定式 Eye-to-Hand 配置中并非必须。

## 2. 材料清单与结构件

### 2.1 铝型材框架

主框架选用 **2020 铝型材**（20mm×20mm 截面，欧标 6mm 槽宽），部分承力节点可升级为 **3030 型材**（30mm×30mm）。型材材质为 6063-T5 铝合金，表面阳极氧化处理。

| 部件 | 规格 | 数量 | 用途 |
|------|------|------|------|
| 2020 铝型材 | 500mm 长 | 4 根 | 框架竖直立柱 |
| 2020 铝型材 | 600mm 长 | 4 根 | 框架水平横梁（前后向） |
| 2020 铝型材 | 400mm 长 | 4 根 | 框架水平横梁（左右向） |
| 2020 铝型材 | 300mm 长 | 2 根 | 顶部相机横梁 (cam1/cam3) |
| 2020 铝型材 | 250mm 长 | 2 根 | 侧向相机支架 (cam0/cam2) |
| 3030 铝型材（可选） | 500mm 长 | 2 根 | 底部承重横梁 |

### 2.2 连接件与紧固件

- **T 型螺母 (T-Slot Nuts)**：M5 规格，材质碳钢镀锌，装入型材槽内可沿槽滑动，用于任意位置的螺栓紧固。推荐选用弹片预紧型（带弹簧钢球），防止垂直安装时螺母滑落。用量约 40~60 个。
- **内六角螺栓**：M5×12mm 及 M5×16mm，材质 304 不锈钢或 12.9 级合金钢。用量约 60~80 个。
- **角码 (Corner Brackets)**：L 形 2020 铝合金角码，40mm×40mm，厚 3mm，每节点 2 个（对称安装）。用量约 16~20 对。
- **加强三角撑 (Gusset Plates)**：60mm 直角三角板，厚 4mm，M5 螺栓紧固，用于立柱与横梁节点加固，抵抗机器人运行引起的低频振动（< 20 Hz）。用量约 8~12 片。
- **端盖**：2020 型材塑料端盖，仅用于美观和边缘防护，非结构件。

### 2.3 支撑固定件

根据安装场景选择以下方案之一：

1. **重型三脚架适配器**：定制铝合金转接板，一面通过 M6 螺栓连接三脚架的 3/8"-16 云台螺丝，另一面通过 T 型螺母固定在 2020 型材底部。适用于原型验证或临时部署。
2. **天花板/顶棚吊装**：使用 M8 化学锚栓将框架顶部的 3030 型材直接固定于建筑结构。此方案用于长期产线部署，完全隔离机器人与相机平台的机械耦合。
3. **工作单元围栏螺栓固定**：使用 M10 贯穿螺栓将框架立柱直接锁定在 FANUC 工作单元的安全围栏钢结构上。此为推荐部署方案。

### 2.4 线缆管理

- **USB 3.0 延长线**：4 条，每条长度 3~5m，必须选用 **主动式 (Active)** 延长线（内置信号中继芯片），否则无源线缆超过 1.5m 后信号衰减将导致深度数据丢帧。推荐 Startech 或 Delock 品牌，USB-A 公对母。
- **理线夹/扎带**：自粘式理线夹（粘贴于铝型材侧面）用于固定 USB 线沿型材槽走线；尼龙扎带（100mm 规格）用于捆扎线束。
- **蛇皮管/波纹管**（可选）：直径 16mm 分体式波纹管，包裹四根 USB 线束，保护线缆免受机器人油雾和切屑损伤。

### 2.5 电源与集线器

- **有源 USB 3.0 Hub**：7 端口，至少 4 端口支持 USB 3.0 SuperSpeed (5Gbps)，带独立 12V/5A 电源适配器。推荐选用工业级 Hub（如 StarTech ST7300USB3B），确保长时间运行不掉线。四台 D435 通过 Hub 上联到一台工控机。
- **工控机**：推荐配置 Intel i7-12700 及以上，32GB RAM，至少双 USB 3.0 主控制器（Intel 芯片组原生 + 第三方 ASMedia/Renesas），SSD 1TB NVMe。操作系统 Ubuntu 20.04/22.04 LTS。

## 3. CAD 设计与 3D 打印组件

### 3.1 相机安装座

相机安装座是在 SolidWorks（或 Fusion 360）中建模的定制件，由于需要精密配合 D435 相机外壳几何以及 1/4"-20 相机螺纹，无法使用标准铝型材配件直接安装。

**相机座设计要点**：

- **安装面**：与 D435 底部平面配合，预留 1/4"-20 相机螺丝通孔（居中）及两个 M3 定位销孔（防旋转）。由于 D435 外壳底部有两个 M3 螺纹孔（间距 12.5mm），可利用这两个螺孔实现抗扭定位。
- **角度调节机构**：相机座后端设计弧形滑槽（R=50mm，槽宽 5.2mm），允许相机绕水平轴 ±30° 范围内俯仰调节。滑槽内穿 M5 螺栓配合尼龙防松螺母，调定角度后锁紧。
- **转接面**：相机座顶部/侧面预留 M5 通孔阵列（间距 20mm，匹配 2020 型材 T 槽中心距），直接通过 T 型螺母 + M5 螺栓连接到铝型材横梁。

### 3.2 CAD 文件目录

所有 CAD 设计文件位于 `hardware/cad_exports/` 目录：

| 文件名 | 格式 | 内容说明 |
|--------|------|----------|
| `装配体2.STEP` | STEP AP214 | 完整四相机平台装配体（含型材框架、相机座、连接件） |
| `标定连接板.STEP` | STEP | ChArUco 标定板连接件——标定阶段将标定板固定于工件所在位置，精确恢复相机外参 |
| `三角形.STEP` | STEP | 三角形加强支架——用于立柱与横梁的 L 形节点外侧加固 |
| `十字架形.STEP` | STEP | 十字交叉支架——用于顶部四根横梁的中心汇集节点 |
| `标定板.DWG` | DWG | ChArUco 标定板 CAD 加工图纸（包含 7×5 标记阵列、边框尺寸、打印精度要求） |
| `板材.DWG` | DWG | 底板上板材机械加工图纸 |
| `标定连接板.3mf` | 3MF | 标定连接板的 3D 打印文件（含支撑/填充设置） |

原始 SolidWorks 源文件（`.sldprt` 零件文件、`.sldasm` 装配体文件）可根据需求提供。

### 3.3 3D 打印参数

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 材料 | PLA+ 或 ABS | PLA+ 刚性好，ABS 耐油雾环境更优 |
| 填充率 | 40% 六边形填充 | 兼顾重量和抗弯强度 |
| 壁厚 | 3 层（≥1.2mm） | 确保 M5 螺栓紧固不拉脱 |
| 层高 | 0.2mm | 200mm/s 打印速度下的推荐设置 |
| 支撑 | 仅弧形滑槽处需支撑 | 其余部分无悬垂结构 |
| 后处理 | M5 螺纹孔攻丝 | 打印后手动攻 M5 螺纹，如用自攻螺丝可直接拧入 |

## 4. FANUC 机器人工作单元安装方案

### 4.1 Eye-to-Hand 配置原理

本系统采用 **Eye-to-Hand**（眼在手外）配置：四相机平台作为独立于机器人的固定传感器模块，持续观察机器人工作空间。与 Eye-in-Hand（相机安装在机械臂末端）相比，本方案的优劣如下：

**优势**：
- 相机外参仅需一次性标定，无需随机器人运动反复修正
- 视野固定，不存在运动模糊和视野动态变化的问题
- 标定的相机坐标系通过手眼标定可直接固定在机器人基坐标系下

**约束**：
- 工件必须放置于相机视野的有效覆盖区域内
- 机器人本体可能遮挡部分视野（可通过多相机互补缓解）

### 4.2 物理安装

框架通过 **4 个 M10 贯穿螺栓** 直接锁定在 FANUC 工作单元的安全围栏（或独立钢制支架柱）上。安装高度使相机平台的底部横梁距离地面约 1.8~2.2m，俯视工作台上的工件。

- **工作距离**：相机至工件中心的典型距离为 0.8~1.2m（D435 在此距离内深度 RMS 误差 < 2mm@1m）。
- **减振措施**：框架与围栏之间加装 **橡胶减振垫片**（邵氏硬度 A50、厚 5mm），衰减机器人急停或高速运动时经地面传入框架的高频振动。

### 4.3 四相机空间布局策略

```
                    ┌──────────────────────┐
                    │       cam1           │
                    │   (工件前上方)         │
                    │   tilt: -15°         │
                    └──────────────────────┘
     ┌──────────────┐         ┌──────────────┐
     │    cam2      │  工件   │    cam3      │
     │  (左侧俯视)   │  ┌───┐ │  (右侧俯视)   │
     │  tilt: -30°  │  │   │ │  tilt: -30°  │
     └──────────────┘  └───┘ └──────────────┘
                    ┌──────────────────────┐
                    │       cam0           │
                    │   (参考相机, 正上方)   │
                    │   tilt: -45°         │
                    └──────────────────────┘
```

**各相机角色与定位依据**：

- **cam0（参考相机，Reference Camera）**：作为多相机外参标定的全局坐标原点。通常安装在工件正上方或前上方，拥有最好的工件主表面视角。其坐标系即为后续所有处理的全局参考系。俯仰角设为 -45°（向下倾斜 45°），确保视野覆盖工件顶部及前面。
- **cam1**：位于工件前方偏上位置，与 cam0 视场重叠 >30%。用于补偿 cam0 对工件前立面下部的覆盖不足。
- **cam2 和 cam3**：分别位于工件左前方和右前方，与 cam0/cam1 成 ~60°~90° 角度间隔。左右对称布局确保工件侧面特征（凹陷、孔洞、凸缘）全部被至少两台相机同时捕获。其深度数据与 cam0 的重叠区域足以支撑 ICP 点对点配准。
- **视野重叠要求**：相邻相机之间的公共可见区域应 >30% 以上。这是 multi-way ICP 和 TSDF 融合的收敛基础。重叠不足将导致配准退化至局部最优甚至发散。
- **环境光约束**：四相机均需避免 **直射阳光**（通过窗户或天窗）进入相机 FOV。阳光中的红外成分会淹没 D435 投射的红外点阵图案，导致立体匹配失效、深度图出现大面积空洞。建议工作单元四周加装遮光帘或使用红外阻断膜。

## 6. USB 带宽与数据传输设计

### 6.1 带宽需求分析

四台 D435 同时以 1280×720@15FPS 运行时的数据传输量估算：

单帧深度图数据量（1280×720, 16-bit per pixel）：
```
1280 × 720 × 2 bytes = 1,843,200 bytes ≈ 1.84 MB
```

单帧彩色图数据量（1280×720, RGB24）：
```
1280 × 720 × 3 bytes = 2,764,800 bytes ≈ 2.76 MB
```

单帧总数据量 ≈ 4.6 MB。15 FPS 下单相机带宽：
```
4.6 MB × 15 = 69 MB/s
```

考虑到 USB 协议开销（~15%）和 SDK 内部缓冲，单相机实际占用带宽约 **80~100 MB/s**。四台相机总计约 **320~400 MB/s**。

虽然这一数值在 USB 3.0 的理论带宽（5 Gbps ≈ 625 MB/s）以内，但实际瓶颈在于：

1. **USB 主控制器带宽**：Intel / AMD 主板通常提供 1~2 个独立 USB 3.0 主控制器（xHCI）。单个 xHCI 控制器的有效持续吞吐量约为 350~400 MB/s（由于共享 PCIe Gen2×1 上行带宽限制）。四台 D435 合并数据流极易击穿单控制器上限。
2. **USB Hub 共享带宽**：所有下行端口共享同一个上行接口，Hub 本身成为瓶颈节点。

### 6.2 推荐带宽分配方案

**方案 A（推荐：降分辨率）**：
将深度与彩色分辨率从 1280×720 降至 848×480@15FPS。单相机带宽降至约：
```
848 × 480 × 2 × 15 ≈ 12.2 MB/s (depth)
848 × 480 × 3 × 15 ≈ 18.3 MB/s (color)
合计 ≈ 30.5 MB/s ×1.15(协议开销) ≈ 35 MB/s
```
四台总计 ≈ 140 MB/s，单个 xHCI 控制器可稳定承载。

**方案 B（高精度需求：双控制器）**：
若必须保持 1280×720 分辨率，则需使用 **两个独立 USB 3.0 控制器**，每个控制器接两台 D435。这可通过以下方式实现：
- 主板有双 xHCI 控制器（服务器级或高端消费级主板）
- 加装 PCIe USB 3.0 扩展卡（如 Renesas uPD720202 芯片），确保扩展卡使用独立 PCIe 通道（不与主板 xHCI 共享）

### 6.3 USB 线缆与 Hub 选型细则

| 组件 | 要求 | 推荐型号 |
|------|------|----------|
| 相机直连线 | USB 3.0 Type-C to Type-A，长度 1m，带屏蔽编织层 | Intel 原装线缆 |
| 有源延长线 | USB 3.0 Active Repeater Cable，支持 5Gbps，长度 5m | StarTech USB3AUB5M 或 Delock 83474 |
| USB Hub | 7 端口，USB 3.0，独立供电 12V/5A（60W），每端口过流保护 | StarTech ST7300USB3B（工业级） |
| 上联线 | USB 3.0 Type-B to Type-A，长度 1m | Hub 原装配线 |

**注意事项**：
- 严禁使用无源 USB 3.0 延长线超过 1.5m：USB 3.0 SuperSpeed 信号的奈奎斯特频率约为 2.5 GHz，无源线缆在 2m 处的插入损耗可达 -8dB 以上，导致码间干扰和链路降级至 USB 2.0 模式。
- 所有 USB 接头处使用 **热缩管包裹** 或 **自粘硅胶带** 密封，防止工业环境中的油雾水汽渗入导致接触不良。
- 建议在工控机侧对各相机 USB 端口进行 `usb-devices` 枚举记录，确保每次开机后 `/dev/video*` 设备索引与物理相机一一对应。

## 7. 维护与操作注意事项

### 7.1 日常检查项

- 检查四台 D435 的 USB 连接状态是否均为 SuperSpeed（非 High Speed），可通过 `lsusb -t` 查看设备树速率标识。
- 检查相机安装座的 M5 锁紧螺栓是否有松动迹象（建议用油漆笔做标线便于视觉巡检）。
- 开机后拍摄一帧 RGB 图像送检：确认四台相机画面均清晰、无雾化（镜头污染）、无异常位移（外参破坏）。

### 7.2 重新标定触发条件

以下情况需要重新执行 ChArUco 多相机标定（`calib/` 模块）：

- 任一相机安装座螺栓松动并重新紧固
- 型材框架因碰撞产生可见位移
- 更换 D435 相机后（不同相机个体的内参不同）
- 累计运行超过 500 小时（预防性重标定）
