# 喷涂轨迹规划技术文档

## 概述

本文档详细描述多相机三维重建机器人喷涂系统中喷涂轨迹规划模块的完整算法流程。系统接收从多视角图像重建得到的点云数据（PCD格式），经过几何计算、曲线平滑、姿态生成与速度规划，最终输出FANUC机器人控制器可直接执行的LS格式程序文件。整个过程涉及凸包计算、切片相交、B样条平滑、姿态插值及曲率自适应速度规划等关键步骤。

---

## 1. 凸包计算与最小有向包围盒提取

### 1.1 输入数据

输入为经过多相机系统重建得到的三维点云，存储为PCD文件格式，坐标单位为毫米（mm）。点云经过去噪与下采样预处理，密度与精度满足后续几何计算的精度要求。

### 1.2 凸包计算

使用Open3D库的`compute_convex_hull`方法对点云进行凸包计算。该方法的底层实现为Qhull算法，参数`joggle_inputs=True`用于处理共面点与数值精度问题：当输入点云存在近似共面的点时，Qhull会通过微小扰动（joggle）打破退化配置，避免计算失败或产生不稳定的输出。

凸包输出的结果为三角网格，由顶点列表和三角形索引列表组成。每个三角形由三个顶点索引定义，其法向量由右手定则确定，指向凸包外侧。凸包作为原始点云的紧致外包络，在保证几何完整性的前提下大幅简化了后续的平面求交计算。

### 1.3 最小有向包围盒提取

在凸包网格基础上，计算最小有向包围盒（Oriented Bounding Box, OBB）。OBB由中心点坐标、三个正交轴向的单位向量及三个轴向上的半边长（extent）组成。OBB的计算采用主成分分析（PCA）方法：对凸包顶点进行协方差分析，获取三个主方向作为包围盒的局部坐标轴。

### 1.4 底部轴判定

在喷涂应用中，通常需要将工件的底面排除在外，因为底面一般为支撑面或非喷涂区域。底部轴的判定方法为：将OBB的三个局部坐标轴分别投影到世界坐标系Z轴（重力方向）上，取投影绝对值最大的轴作为"底部轴"（bottom axis）。该轴对应包围盒中与世界竖直方向最为接近的方向，其所对应的底面将在后续切片与偏移计算中被裁剪排除。

## 2. 切片策略与偏移距离计算

### 2.1 切片轴选择

系统支持两种切片方向选择模式：

**自动模式（默认）**：根据OBB的几何特征自动选取两个切片轴。纵向轴选取与世界Z轴夹角最小的OBB局部轴，即底部轴；横向轴选取OBB剩余两轴中在水平面上投影较长的轴，确保横向切片覆盖更广的工件轮廓范围。两个切片轴共同构成一组平行平面族的法方向。

**手动模式**：允许用户通过配置文件显式指定切片方向，可选值为`"vertical"`（竖直方向）、`"x"`（世界X轴）、`"y"`（世界Y轴）、`"z"`（世界Z轴）。手动模式适用于工件姿态已知且需要精确控制喷涂路径走向的场景。

### 2.2 切片间距与平面求交

配置参数`SLICE_SPACING=20mm`定义相邻切片平面之间的间距。对于每个切片轴，以包围盒在该轴上的范围为界，从最小端开始以20mm步长生成一组等间距的切割平面。

每个切片平面与凸包三角网格求交的过程为：遍历凸包的所有三角形，判断平面与三角形是否相交，若相交则计算交线段（平面与三角形两边各有一个交点）。每条交线段记录两个端点。单个切片平面上所有交线段构成该截面的轮廓多边形边界。

### 2.3 交点排序

一个切片平面可能与凸包产生多个分离的交线段（处理非凸截面的情况），亦可能产生连续闭合的轮廓。将所有交线段的端点投影到切片平面的局部二维坐标系中，以截面点的几何中心为原点进行角度排序（angular sorting），将无序的线段端点组织为有序的闭合轮廓多边形。排序后验证闭合性：首尾点之间的距离应小于预设容差，否则视为开放轮廓并记录警告。

### 2.4 偏移距离与迭代修正

**基础偏移**：配置参数`OFFSET_DISTANCE=80mm`定义喷枪喷嘴到工件表面的标称距离。对于轮廓上的每个交点，沿该点处截面轮廓的局部法向量方向向外偏移80mm，得到初步的轨迹点位置。

**偏移修正**：单纯在二维截面上偏移可能使轨迹点与三维凸包表面之间的距离偏离标称值，尤其是在曲率较大的区域。系统采用迭代修正策略：计算轨迹点到凸包表面最近点的欧氏距离，若与标称偏移距离的偏差超过阈值（默认1.0mm），则沿当前点到最近表面点的连线方向调整轨迹点位置，重复至收敛或达到最大迭代次数（默认10次）。此修正确保轨迹点在三维空间中精确维持指定的喷涂距离。

### 2.5 底面裁切

配置参数`BOTTOM_SKIP=100mm`定义底部区域的裁切高度。对于判定为底部轴的OBB局部方向，在包围盒的最小端面（即底面）以上100mm范围内的所有切片及轨迹点均被剔除。裁切的逻辑依据是：从底面端面起算，沿底部轴方向向上量取指定距离，在该范围内的截面不参与后续轨迹生成。

## 3. B样条平滑与弧长重采样

### 3.1 曲线稠密化

将所有截面生成的偏移轨迹点按截面顺序拼接为一条连续的原始折线。由于相邻截面之间的点间距可能较大（截面间距20mm），直接在此折线上进行平滑会发生欠拟合。因此先进行稠密化处理：沿折线以`DENSIFY_STEP=40mm`的步长进行线性插值，在相邻原始点之间均匀插入中间点，使曲线采样密度满足后续B样条拟合的要求。

### 3.2 B样条平滑

使用SciPy库的`splprep`函数对稠密化后的曲线进行参数化B样条拟合。关键参数设置如下：

- **阶数**（degree）：`k=3`，即三次B样条，保证曲线二阶连续（C²连续），满足机器人运动对轨迹平滑性的要求。
- **平滑因子**（smoothness）：`s=1.5`（单位mm），控制曲线对输入数据的拟合紧密程度。较小的平滑因子使样条更贴近原始数据；较大的平滑因子产生更光滑但可能偏离原始数据的曲线。1.5mm的平滑量在保持轨迹精度的同时有效抑制了重建噪声引起的局部抖动。

`splprep`以曲线的累积弦长作为参数化方式，将三维坐标`(x, y, z)`分别作为参数的函数进行拟合，返回B样条的节点向量与控制点。

### 3.3 弧长重采样

B样条平滑后，使用弧长参数化方式在曲线上进行等距重采样。配置参数`EXECUTE_POINT_SPACING=30mm`定义相邻执行点之间的弧长间距。弧长积分采用数值方法（辛普森积分或自适应高斯积分），沿B样条曲线累积弧长并在指定弧长位置生成执行点。重采样后的点即为机器人末端执行器的实际经过位置。

### 3.4 平滑后距离修正

B样条平滑操作会使轨迹点相对于凸包表面产生微小位移，导致实际喷涂距离偏离标称值。系统在平滑与重采样后再次执行距离修正：对每个执行点，在原始重建点云（非凸包的简化顶点，而是原始稠密表面点）中搜索最近点，沿该方向调整位置以恢复`OFFSET_DISTANCE`的标称间距。原始点云表面的法向量信息较凸包三角面片更为精细，修正精度更高。

## 4. 喷枪姿态生成

### 4.1 喷涂方向计算模式

系统支持四种喷涂方向计算模式，通过配置参数`SPRAY_DIRECTION_MODE`选择：

- **`closest_surface`（默认推荐）**：对每个轨迹点，在原始重构点云中搜索空间最近点，最近点到轨迹点的方向向量即为喷涂方向。此模式使喷枪始终垂直于工件局部表面，喷涂效果最为均匀。
- **`closest_hull`**：与上述类似，但搜索空间限定在凸包表面而非原始点云，计算速度更快但精度略低。
- **`centroid`**：所有轨迹点统一指向点云几何中心，适用于球形或对称工件的均匀包裹喷涂。
- **`nearest_normal`**：使用PCD中最近点的法向量（若PCD包含法向信息）作为喷涂方向，对点云质量要求较高。

### 4.2 工具坐标系构建

喷枪姿态以工具坐标系（Tool Frame）的形式表达。工具坐标系的三个轴按以下规则逐点构建：

1. **Z轴（喷涂轴）**：由喷涂方向向量赋值。配置参数`TOOL_Z_TO_SPRAY_SIGN`（取值`+1`或`-1`）控制Z轴指向与喷涂方向同向或反向，以适配不同喷枪的实际安装朝向。
2. **Y轴（切向轴）**：取轨迹在当前点的切线方向（由B样条弧长参数的一阶导数求得）在垂直于Z轴的平面上的投影方向。Y轴方向与工具横移方向一致。
3. **X轴（侧向轴）**：由Y轴与Z轴的叉积确定，确保为右手坐标系：`X = Y × Z`。

三个轴经施密特正交化保证严格正交性后，组合为3×3旋转矩阵。

### 4.3 半球连续性维护

在相邻轨迹点之间，由于表面曲率变化或噪声干扰，计算出的工具Z轴可能发生接近180度的方向翻转（即工具姿态在两个相反方向之间突变）。这种翻转导致机器人腕关节的剧烈旋转，在实际喷涂中不可接受。

系统采用逐点连续性维护策略：从第二个轨迹点开始，检查当前点Z轴与前一点Z轴的点积符号。若点积为负（角度超过90度），则将当前点的工具坐标系绕X轴旋转180度进行翻正。此处理逐点传播，保证整条轨迹的姿态方向一致性。

### 4.4 最大旋转步长限制

即使维护了半球连续性，相邻点之间的姿态变化仍可能过于剧烈，超出机器人腕关节的角速度能力。配置参数`MAX_ROT_STEP_DEG=8.0`设定相邻两个轨迹点之间允许的最大旋转角度（以度为单位）。若两点之间工具坐标系的旋转角度超过此阈值，则对超限的轨迹点进行姿态平滑：在当前点与前一点的旋转之间进行球面线性插值（SLERP），以使旋转步长缩限至允许范围内。

### 4.5 工具安装偏角补偿

喷枪喷嘴的实际安装方向可能与工具坐标系的标称方向存在固定偏角。配置参数`TOOL_ROT_OFFSET_WPR_DEG`定义绕工具坐标系X、Y、Z轴的旋转偏角补偿量（单位为度），以W/P/R（Yaw/Pitch/Roll）三元组形式指定。此补偿旋转以右乘方式施加于每个轨迹点的姿态矩阵，物理含义为工具末端的固定安装偏角。

### 4.6 欧拉角转换与FANUC约定

轨迹点姿态最终需要表示为FANUC机器人控制器所需的欧拉角格式。FANUC采用固定轴XYZ外旋欧拉角约定（Fixed XYZ Extrinsic），在SciPy中对应的旋转顺序字符串为`'xyz'`（注意：SciPy中使用`'xyz'`表示固定轴外旋，与某些文献中的记法相反）。旋转矩阵经SciPy的`Rotation.as_euler('xyz', degrees=True)`转换为W/P/R三值（即绕固定X、Y、Z轴的旋转角，单位为度）。W/P/R分别对应FANUC机器人程序中的Yaw（绕X）、Pitch（绕Y）、Roll（绕Z）。

## 5. 曲率自适应速度规划

### 5.1 曲率代理指标

为简化曲率计算并提高鲁棒性，系统采用"单位路径长度转角"作为曲率的代理指标（proxy），单位为deg/mm。对相邻轨迹点i和i+1，计算其喷涂方向（Z轴）之间的夹角（度），除以两点之间的弧长（mm），得到该局部段的转角密度：

```
curvature[i] = angle_between(z_axis[i], z_axis[i+1]) / arc_length[i, i+1]
```

该指标反映了工件表面法向沿轨迹变化的剧烈程度：曲率大的区域转角密度高，需要较低的运动速度以保证喷涂质量；平直区域转角密度低，可以提高速度以提升效率。

### 5.2 直段与曲段分类

配置参数`CURV_DEG_PER_MM_TH=0.25`（deg/mm）定义直段与曲段的阈值分界。当局部转角密度低于0.25 deg/mm时判定为直段，高于该值时判定为曲段。按常规逻辑，直段应使用较高的速度以提高效率，曲段应使用较低的速度以保证质量。但本系统根据喷涂工艺的特定要求采用特殊策略：

- **直段速度**：`SPEED_STRAIGHT=100 mm/sec`（较低速度），因直段为工件主要可视面，需保证更高的喷涂均匀度。
- **曲段速度**：`SPEED_CURVE=150 mm/sec`（较高速度），因曲段一般为边缘或过渡区域，对均匀度要求相对宽松。

### 5.3 速度平滑窗口

为避免频繁的速度切换引起机器人运动抖动，系统使用滑动平均窗口对分类结果进行平滑。配置参数`CURV_SMOOTH_WINDOW=5`定义窗口大小：对每个点的前后各两个邻域点（共5点），计算区域内曲段的占比。若占比超过50%则分配曲段速度，否则分配直段速度。窗口平滑有效消除了因局部噪声导致的孤立的、短暂的速度变化。

### 5.4 速度取整

FANUC控制器通常要求速度值为整数。配置参数`SPEED_ROUND=5 mm/sec`定义速度值的取整步长。所有计算出的速度值向上或向下舍入到最近的5 mm/sec的整数倍（即100、105、110等）。

## 6. FANUC LS文件格式

### 6.1 文件结构概述

FANUC机器人使用LS（Ladder/Sequential）格式的ASCII文本程序文件，通过FTP以ASCII模式上传至控制器的`md:/`存储区。LS文件包含多个逻辑段，由以斜杠开头的段标记分隔。

### 6.2 段说明

**`/PROG`段**：程序的头部行，包含程序名称。文件第一行必须为`/PROG <程序名>`，程序名遵循FANUC命名规范（字母开头，不超过规定长度）。

**`/ATTR`段**：程序元数据，包含所有者（OWNER）、创建日期（CREATION DATE）、内存大小预估（MEMORY SIZE）、程序保护属性（PROTECT）等。此段由控制器自动维护，生成时可设置基本值。

**`/APPL`段**：应用设定。对于喷涂应用，需声明`PAINT_PROCESS`以启用喷涂相关的工艺设定。该段包含所用用户坐标系（User Frame, UF）、工具坐标系（Tool Frame, UT）的编号引用、喷涂预设参数（如喷枪开关逻辑、流量等）。

**`/MN`段**：运动指令段，为LS文件的核心内容。每行运动指令的格式为：

```
<行号>: <运动类型> P[<索引>] <速度>mm/sec <定位形式> ;
```

其中：
- 行号从1开始递增。
- 运动类型为`L`（直线运动，Linear）或`J`（关节运动，Joint）。本系统全部采用`L`类型以保证轨迹精度的直线路径。
- `P[i]`索引对应`/POS`段中的第`i`个位置。
- 速度值后紧跟`mm/sec`单位标识。
- 定位形式`CNT`（Continuous）值指定过渡圆角大小：`CNT100`为最大平滑过渡，`CNT0`（即FINE）为精确到位停止。在曲段与直段的过渡处可适当加大CNT值。

**`/POS`段**：位置数据段，定义程序中用到的所有笛卡尔位姿。每个位置记录的格式为：

```
P[i]{
   X = <X值>  mm, Y = <Y值>  mm, Z = <Z值>  mm,
   W = <W值> deg, P = <P值> deg, R = <R值> deg,
   UF = <用户坐标系编号>, UT = <工具坐标系编号>,
   CONFIG = '<配置字符串>'
};
```

其中X/Y/Z为工具中心点在世界坐标系下的位置（mm），W/P/R为固定X-Y-Z外旋欧拉角（deg），UF/UT为坐标系编号，CONFIG字符串编码机器人的关节构型（如`'F U T, 0, 0, 0'`对应Front/Up/Top腕关节构型）。

### 6.3 上传方式

生成的LS文件通过FTP协议上传至FANUC控制器。上传必须使用FTP的ASCII传输模式（`TYPE A`命令），因为LS文件的行尾约定为CRLF（`\r\n`），ASCII模式会自动将Unix系统的LF转换为CRLF。二进制模式上传将导致控制器无法正确解析换行符。目标路径为`md:/`下的对应目录。

### 6.4 多面喷涂的程序组织

对于需要分面喷涂的复杂工件，推荐采用CALL调用式的程序组织方案：为每个喷涂面生成独立的LS子程序文件，各子程序包含该面的完整轨迹与工艺设定。另创建一个主程序，通过`CALL <子程序名>`指令依次调用各面子程序。主程序在调用之间可插入换枪、等待工件旋转等辅助指令。这种组织方式便于调试、修改和维护，单个面的轨迹调整不影响其他面。
