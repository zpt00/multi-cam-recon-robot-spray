## 项目一：基于多目 RealSense 的室内场景三维重建与机器人喷涂路径生成系统

**项目周期**：2025年9月 — 2026年1月（约4个月）

**技术栈**：Python · Open3D · OpenCV · PyRealSense2 · ICP · ChArUo · NumPy/SciPy · B-Spline · Poisson Reconstruction · FANUC TP/LS · FTP/UDP

**项目概述**：
本项目构建了一套完整的"视觉感知 → 三维重建 → 轨迹规划 → 机器人执行"闭环系统。使用 4 台 Intel RealSense D435 深度相机搭建多视角采集平台，实现室内工件的高精度稠密三维重建，并基于重建结果自动生成 FANUC 工业机器人的喷涂轨迹程序（LS 文件），通过 FTP 直接下发至机器人控制器执行，实现了从三维感知到自动化加工的端到端流程。

**负责内容与成果**：

### 1. 多相机标定与外参优化
- 基于 ChArUco 标定板的多相机外参实时标定工具，支持 2~4 台相机任意组合采集，cam0 作为参考坐标系，实时求解各相机到参考相机的变换矩阵（4x4 刚体变换）
- 设计鲁棒外参筛选算法：对多次采集结果按相机维度分别收集，采用**四元数均值 + 中位数平移**计算统计中心，结合 **MAD（Median Absolute Deviation）** 剔除离群值，选出每台相机的最佳外参
- 输出 YAML / NPZ 格式标定结果及 CSV 报告，便于后续追溯与调试
- **成果**：标定精度达到平移误差 <5mm、旋转误差 <1°，满足机器人引导场景要求

### 2. 多视角稠密点云实时融合
- 基于 Open3D 实现 4 台相机的实时深度图采集、彩色图对齐与点云生成
- 设计**双路点云策略**：
  - **稠密点云**（stride=2）：保留细节，用于最终融合与可视化
  - **稀疏 ICP 点云**（stride=8 + 体素降采样）：用于配准，提升匹配稳定性与速度
- 实现 **Point-to-Plane ICP** 微调，每 15 帧对各非参考相机相对参考相机做位姿修正，融合 fitness/rmse 阈值判定，支持累积修正模式
- 可选多级后处理：统计离群滤波、半径滤波、体素空洞补点等
- 实时 Open3D 可视化与交互保存，支持按 s 保存融合点云 / 按 r 重置 ICP / 按 q 退出
- **成果**：实现 4 视角实时融合，融合点云密度达到 ~80 万点/帧，帧率 8~12 FPS

### 3. 工件三维重建与网格化
- 对融合后的稠密点云进行预处理：Z 轴裁剪、体素降采样、统计/半径离群滤波、DBSCAN 聚类去噪
- 采用 **Poisson Surface Reconstruction** 进行曲面重建，结合 Fisher 均匀重采样与法向估计，生成高精度三角网格模型
- 网格后处理：顶点聚类简化、平滑滤波、非流形边/退化三角片清理

### 4. 凸包切片轨迹规划
- 设计基于**凸包 + OBB（Oriented Bounding Box）**的环绕式轨迹生成算法：
  - 计算工件点云的凸包网格，获取最小有向包围盒 OBB
  - 自动识别竖直方向与水平长轴作为切方向，沿 OBB 局部坐标轴等距切片（间距 20mm）
  - 对切片多边形做法向偏移（standoff 80mm），生成喷枪远端轨迹
  - 底部自动避让（clearance 100mm），避免碰撞
- B 样条曲线平滑 + 弧长重采样，保证轨迹连续性
- 通过 **RaycastingScene** 或 KDTree 查询轨迹点到凸包表面的最近距离，迭代优化确保偏移一致
- **成果**：支持任意复杂形状工件的全自动轨迹生成，平均轨迹点 200~500 个

### 5. 喷枪姿态生成与机器人代码输出
- 实现 **closest_surface** 喷枪定向模式：轨迹点指向最近原始点云表面点，确保喷枪始终对准工件表面
- 工具姿态约束：切向 Y 轴投影 + Z 轴朝向工件 + 右手定则 X 轴，限制最大转角步长（8°/步），防止机器人腕部翻转
- 曲率自适应速度规划：直线段 100mm/s，弯道段 150mm/s，曲率平滑窗口避免频繁变速
- 生成 FANUC 标准 .LS 程序文件，包含完整的位置数据（X/Y/Z/W/P/R）、速度指令、用户坐标系/工具坐标系配置
- 通过 **FTP** 自动上传至 FANUC 机器人控制器，支持 UDP 握手协议与 PLC 协同控制
- **成果**：实现从点云采集到机器人程序下发的全自动流程，单次任务 3~5 分钟完成

### 6. 系统集成与部署
- 整套系统在 Ubuntu / Windows 双平台验证，实际部署于 FANUC 工业机器人喷涂产线
- 机器人手眼标定模块（eye-to-hand），将重建坐标系统一到机器人基坐标系
- 支持循环工作模式：UDP 接收 PLC 启动信号 → 自动处理最新 PCD → 上传 LS → 清理中间文件 → 回复完成状态
- **成果**：系统经实际产线测试验证，稳定运行，喷涂路径覆盖率 >90%，重复定位精度 ±3mm

---

**项目亮点总结**：
- 全流程自研，从多传感器标定到机器人执行全覆盖
- 双路点云 + ICP 微调保证融合精度和实时性
- 凸包切片 + B 样条平滑 + 最近表面定向的环绕轨迹规划方法，适应复杂工件
- 深度耦合机器人控制（FANUC LS + FTP + UDP/PLC），工程落地能力强

## 项目二（配合使用）：基于 NVIDIA Isaac Sim 的机器人仿真与运动规划验证

**项目周期**：2026年1月 — 至今

**技术栈**：NVIDIA Isaac Sim · Python · ROS2 · USD · PyTorch

**项目概述**：
利用 NVIDIA Isaac Sim 仿真平台搭建机器人工作站的数字孪生环境，用于验证三维重建与喷涂轨迹生成算法的有效性，降低现场调试风险。

**负责内容**：
- 搭建包含 FANUC 机器人模型、工件、传送带、传感器在内的完整仿真场景
- 导入真实重建的工件网格模型，在仿真环境中复现喷涂路径规划结果
- 验证机器人运动学可达性、碰撞检测与轨迹平滑度
- 探索基于 Isaac Sim 的合成数据生成，用于训练视觉感知模型

**成果**：
- 实现真实采集点云 → 仿真环境验证的闭环，提前发现 3 处轨迹碰撞风险
- 仿真轨迹与实机运行轨迹偏差 <5mm

> 1. 如果被问到"这个项目是你一个人做的吗"→ "核心算法和系统架构是我独立完成的，采集和部署环节有其他同学/同事配合"
> 2. 如果被问到"ICP 为什么只对非参考相机做"→ "参考相机（cam0）是基准，所有非参考相机都对齐到它，如果所有相机都对等做 ICP 会导致整体漂移"
> 3. 如果被问到"为什么用凸包切片而不是直接对网格做路径"→ "凸包切片计算量更小，对点云噪声的鲁棒性更强，而且天然适用于环绕式喷涂这种需要整体包络的工艺"
> 4. 如果被问到"你的角色"→ "我是项目负责人，从方案设计到编码实现再到现场部署全程参与"
