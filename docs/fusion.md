# 点云融合与滤波技术文档

## 概述

本文档详述多目 RealSense D435 三维重建系统中点云融合与滤波子系统的技术实现。该子系统负责将 2~4 台深度相机采集的 RGB-D 数据，通过外参变换、ICP 精配准、TSDF 体素融合以及时空域滤波等手段，生成高精度稠密彩色点云与三角网格模型，为后续的轨迹规划与机器人喷涂提供几何基础。

整体融合管线分为两条路径：

- **实时在线融合**（`multi_d435_fusion_dense_icp.py`）：以 8~15 FPS 持续运行，适用于在线预览与实时调整。
- **离线批量融合**（`multi_d435_tsdf_batch_icp.py`）：采集多帧稳定 RGB-D 后做 TSDF 体素融合，输出最高质量的点云与网格，适用于最终建模。

下文从双路点云策略、Point-to-Plane ICP、TSDF 体素融合、时空域滤波以及性能权衡五个方面展开论述。

---

## 1. 双路点云策略：稠密与稀疏分离

### 1.1 问题分析

在多相机实时融合场景中，点云同时承担两种角色：

1. **最终输出**：用于可视化、保存、网格重建的点云，要求尽可能高的点密度以保留工件表面细节。
2. **ICP 配准**：用于估计各非参考相机相对参考相机的位姿修正矩阵，要求点云稀疏且均匀分布以保证匹配速度和收敛稳定性。

如果仅使用单一稠密点云承担两种职责，会引入两个矛盾：稠密点云配准计算开销极大（点对搜索复杂度 O(N log N)），而稀疏点云输出会丢失几何细节。因此，系统设计了**双路点云分离策略**。

### 1.2 实现方案

在 `multi_d435_fusion_dense_icp.py` 中，每台相机每帧同时生成两套点云：

| 点云类型 | 变量名 | 采样步长 | 体素降采样 | 用途 |
|---------|--------|---------|-----------|------|
| 稠密点云 | `dense_pcds` | stride=2 | 默认无（`DENSE_RAW_VOXEL_SIZE=0.0`） | 最终融合、显示、PLY 保存 |
| ICP 稀疏点云 | `icp_pcds` | stride=8 | 0.01m 体素降采样 | 仅用于 ICP 匹配估计位姿修正 |

采样步长的定义：在深度图生成点云时，仅按步长间隔取像素坐标。stride=2 表示每 2 个像素采样一次，即水平方向取 1/2、总点数约为全分辨率（stride=1）的 1/4。stride=8 表示每 8 个像素采样一次，总点数约为 stride=2 的 1/16。

体素降采样（Voxel Downsample）进一步对 ICP 点云做空间均匀稀疏化：将空间划分为边长 0.01m 的立方体格，每个格内仅保留一个点（质心），从而在降低点数的同时维持空间分布的均匀性，这对 ICP 匹配的稳定性至关重要。

### 1.3 位姿合成与稠密变换

ICP 从稀疏点云计算得到各非参考相机相对 cam0 的修正矩阵 `T_icp_refine`，然后合成的总变换矩阵为：

```
T_total = T_icp_refine @ T_init
```

其中 `T_init` 为 ChArUco 标定获得的初始外参（camera → cam0），`T_icp_refine` 为 ICP 估计的增量修正。该合成矩阵 `T_total` 被应用于稠密点云做空间变换，因此最终输出的融合点云保留了稠密细节，而 ICP 的运算量仅由稀疏点云决定。

这种分离设计将配准的计算开销与输出点云的质量完全解耦：即使稠密点云达到数十万至百万点级别，ICP 的每次匹配仅处理数千至数万个稀疏点。

## 2. Point-to-Plane ICP 精配准

### 2.1 两种 ICP 变体的数学本质

ICP（Iterative Closest Point）的核心是在每次迭代中建立源点云与目标点云之间的对应关系，并求解使对应点对距离最小化的刚体变换。

**Point-to-Point ICP** 最小化欧几里得距离：

```
E(R, t) = Σ_i || (R·p_i + t) - q_i ||^2
```

其中 p_i 为源点，q_i 为目标点云中 p_i 的最近点。

**Point-to-Plane ICP** 最小化源点到目标点局部切平面的投影距离：

```
E(R, t) = Σ_i [ (R·p_i + t - q_i) · n_i ]^2
```

其中 n_i 是目标点 q_i 处的法向量。该误差度量衡量的是源点沿目标表面法向方向的偏离，而非绝对三维距离。

### 2.2 收敛性优势

Point-to-Plane 在以下条件下收敛更快、更稳定：

- 两帧点云之间存在平移偏差，但表面大体共面（即两个点云采样的是同一物理表面）；
- 目标表面局部可近似为平面。

在多相机融合场景中，各相机从不同视角观测同一工件表面，经过 ChArUco 标定后初始位姿偏差通常在毫米级，因此表面局部近似平面的假设高度成立。Point-to-Plane 在此条件下能有效区分沿表面方向的滑移（sliding）与沿法向的穿透（penetration），避免点对错误关联导致的配准漂移。

### 2.3 法向量估计

Point-to-Plane ICP 要求源点云和目标点云均具有法向量。系统使用 Open3D 的 `estimate_normals` 接口，采用混合 KD 树搜索参数：

```python
o3d.geometry.KDTreeSearchParamHybrid(radius=0.03, max_nn=30)
```

该参数表示：在半径 0.03m 范围内搜索，最多取 30 个最近邻点，对邻域点做 PCA 求最小特征值对应的特征向量作为法向量方向。混合搜索确保稀疏区域也能找到足够邻域点计算法向。

### 2.4 参数配置

ICP 配准的关键参数如下：

| 参数 | 值 | 说明 |
|-----|-----|------|
| `max_correspondence_distance` | 0.03m | 最大对应点距离阈值，超过此距离的点对不参与优化 |
| `max_iter` | 30 | 最大迭代次数 |
| `fitness_threshold` | 0.15 | 内点率下限，低于此值拒绝本次 ICP 结果 |
| `rmse_threshold` | 0.02m | 内点 RMSE 上限，超过此值拒绝本次 ICP 结果 |
| `RUN_ICP_EVERY_N_FRAMES` | 15 | 每 15 帧执行一次 ICP，降低计算负载 |

### 2.5 累积修正策略

系统支持两种 ICP 修正模式，由 `ENABLE_ICP_ACCUM` 控制：

- **逐帧独立修正**（`ENABLE_ICP_ACCUM=False`）：每次 ICP 直接得出新的 `T_icp_refine = T_frame`，不依赖历史结果。适用于相机位姿频繁变化的场景。
- **累积修正**（`ENABLE_ICP_ACCUM=True`）：`T_icp_refine = T_frame @ T_icp_refine`，每次 ICP 结果左乘到已有修正矩阵上。适用于相机位姿固定、仅需逐步收敛到最优外参的场景。经验表明，累积模式在静态多相机设置下噪声更小、收敛更稳。

## 3. TSDF 体素融合

### 3.1 设计动机

实时融合（第 1 节方案）的优势是即时反馈，但单帧点云拼接的致命弱点是帧间深度噪声无法通过多帧平均消除，且点云拼接在重叠区域存在明显的"重影"效应（同一表面被多视角重复采样，拼接后形成双层点云）。TSDF（Truncated Signed Distance Function）体素融合从根本上解决了这一问题。

### 3.2 批量采集流程

`multi_d435_tsdf_batch_icp.py` 采用离线批量模式：

1. **预热阶段**（WARMUP_FRAMES=60 帧）：启动相机后丢弃前 60 帧，等待曝光和白平衡稳定。
2. **批量采集**（CAPTURE_FRAMES=10 帧）：每台相机采集 10 帧 RGB-D，帧间间隔 0.5 秒，期间工件保持静止。
3. **ICP 精配准**：将每台相机的 10 帧合并为一个累计 ICP 点云（stride=4，voxel=0.015m），用累计点云做一次 ICP 配准，得到相对于 ChArUco 标定结果的修正矩阵。
4. **TSDF 融合**：将所有相机的全部 RGB-D 帧整合到统一体素空间。

### 3.3 Open3D ScalableTSDFVolume 参数

| 参数 | 值 | 说明 |
|-----|-----|------|
| `voxel_length` | 0.005m | 体素边长 5mm，控制输出分辨率 |
| `sdf_trunc` | 0.04m | SDF 截断距离，表面外 4cm 以上的体素不更新 |
| `color_type` | RGB8 | 启用带颜色的 TSDF 重建 |

### 3.4 坐标系转换

TSDF 体素空间定义在参考相机（cam0）坐标系下。`integrate` 接口接收的 `extrinsic` 参数为 world→camera 的变换，即参考相机到当前相机的变换矩阵。因此对每台相机的 `T_total`（camera→cam0）取逆：

```python
extrinsic_ref_to_cam = np.linalg.inv(T_total[name])
```

### 3.5 后处理

融合完成后对输出做两级后处理：

- **统计离群点移除**（Statistical Outlier Removal）：`nb_neighbors=20`，`std_ratio=2.0`。对每个点计算到 k 近邻的平均距离，排除平均距离超过 2 倍全局标准差的离群点。
- **网格平滑**（Mesh Smoothing）：`filter_smooth_simple`，迭代 1 次。采用拉普拉斯平滑，将每个顶点向其邻域顶点的平均位置移动。

### 3.6 TSDF 与直接融合的对比

| 维度 | 直接点云拼接 | TSDF 体素融合 |
|-----|------------|-------------|
| 重叠区域处理 | 双层点云叠加，存在重影 | 多帧深度加权平均，表面唯一 |
| 噪声抑制 | 依赖离群滤波后处理 | 多帧观测内生平均，天然去噪 |
| 输出形式 | 点云 | 点云 + 水密三角网格 |
| 运行模式 | 实时在线 | 离线批量 |
| 薄壁结构 | 不丢失 | 可能因 SDF 截断而丢失（<trunc 距离的薄壁） |
| 计算开销 | 低，每帧 O(N) | 高，积分耗时与体素数量成正比 |

## 4. 时空域联合滤波

`spatial_temporal_filter.py` 实现了四种深度滤波模式，用于在单相机深度采集阶段进行预处理，提升原始深度图质量后再生成点云。

### 4.1 空间滤波

所有模式共享同一空间滤波器：OpenCV 的 `bilateralFilter`（双边滤波），参数如下：

| 参数 | 值 | 说明 |
|-----|-----|------|
| `d` | 9 | 滤波核直径（像素） |
| `sigmaColor` | 10mm | 颜色/深度域标准差，深度差超过 10mm 的邻域像素权重显著降低 |
| `sigmaSpace` | 7px | 空间域标准差，距离超过 7 像素的邻域像素权重显著降低 |

双边滤波的优势在于同时考虑空间邻近性和深度相似性，能在平滑深度噪声的同时保留深度边缘（即物体轮廓边界的深度跳变不被模糊）。

### 4.2 `"original"` 模式：纯 EMA 平滑

对每个持续有效的像素，执行指数移动平均：

```
depth_accum[t] = alpha * depth_new[t] + (1 - alpha) * depth_accum[t-1]
```

其中 `alpha = 0.05`。该模式假设表面静止，所有像素深度值应当趋于一致，通过长时域的指数衰减平滑随机噪声。但其缺点是当相机或物体移动导致深度跳变时，EMA 会产生拖尾伪影——旧值在新表面出现后仍持续数个帧周期才逐渐收敛。

### 4.3 `"jump"` 模式：跳变检测

针对 EMA 拖尾问题的改进。核心逻辑：

1. 计算当前帧与累积帧的深度差：`diff = |depth_new - depth_accum| * depth_scale`。
2. 若 `diff >= JUMP_THRESHOLD`（0.05m），判定该像素发生了表面切换（跳变），直接替换为新值：`depth_accum = depth_new`。
3. 若 `diff < JUMP_THRESHOLD`，判定为同一表面的随机噪声，执行 EMA 平滑。

该模式在深度变化检测的可视化叠加层中以红色高亮标出跳变像素，便于调试。

### 4.4 `"edge"` 模式：边缘规避

核心思路：利用 Sobel 算子检测深度图中的几何边缘（物体轮廓、表面转折处），对边缘像素做直接替换，对平坦区域做 EMA 平滑。

边缘检测流程：

1. 对深度图做高斯模糊（ksize=5），抑制噪声对梯度计算的影响。
2. Sobel 算子计算 x 和 y 方向梯度，合成梯度幅值。
3. 梯度幅值 `> EDGE_THRESHOLD`（默认 0.035m/px）的像素标记为边缘。
4. 对边缘掩码做形态学膨胀（dilate，核大小 3），扩大边缘区域覆盖范围。

该模式特别适用于保持高频几何特征（如棱角、孔洞边缘）的同时平滑大块平坦表面。

### 4.5 `"realsense"` 模式：Intel SDK 滤波链

直接调用 Intel RealSense SDK 内置的时空滤波器：

```
spatial_filter (magnitude=2, alpha=0.5, delta=20)
    → temporal_filter (alpha=0.4, delta=20)
```

RealSense 的 `spatial_filter` 使用基于边缘保持的域变换滤波，`temporal_filter` 使用带指数衰减的帧间平滑，内部包含了像素有效性检测和跳变抑制逻辑。该模式的优势是经 Intel 官方优化，在各种硬件上运行稳健；缺点是参数调节的灵活性不如自定义实现。

### 4.6 附加 RealSense 滤波器

在 `multi_d435_fusion_dense_icp.py` 中可选的 RealSense 后处理滤波器链：

- **Decimation Filter**（默认关闭）：降采样深度图分辨率，减小后续处理负担，但会损失细节。
- **Spatial Filter**（magnitude=2，alpha=0.5，delta=20）：边缘保持的域变换空间平滑。
- **Temporal Filter**：帧间时序平滑，需为每台相机独立实例化（因为时序滤波器维护内部历史状态）。
- **Hole Filling Filter**（mode=1，farest from around）：用周围最远有效深度值填充空洞。注意这不适合高精度尺寸测量，因为填充值是估计值而非真实测量。

## 5. 性能指标与参数权衡

### 5.1 采样步长 vs. 点云密度

以 1280x720 分辨率、深度有效区域约 60% 估算：

| stride | 理论最大点数 | 实际有效点数（估算） | 相对密度 |
|--------|------------|-------------------|---------|
| 1 | 921,600 | ~550,000 | 1x（基准） |
| 2 | 230,400 | ~140,000 | ~0.25x |
| 4 | 57,600 | ~35,000 | ~0.06x |
| 8 | 14,400 | ~8,600 | ~0.016x |

stride 1 到 stride 2 的点数下降约 4 倍，但视觉质量下降几乎不可感知——因为相邻像素对应的三维点间距为亚毫米级（在 50cm 距离处约 0.3mm），远小于实际应用需求的 1~2mm 精度。

### 5.2 ICP 加速比

ICP 每次迭代的计算量主要由对应点搜索（KDTree 最近邻查询）决定，复杂度为 O(N log M)。相比于使用稠密点云做 ICP，双路策略的加速效果来自两点：

- **stride 从 2 到 8**：点数减少约 16 倍，KDTree 查询减少约 16 倍。
- **体素降采样 0.01m**：进一步减少点数，同时提升空间分布均匀性，改善收敛稳定性。

综合实测，ICP 从稠密模式切换到双路模式后，单次 ICP 耗时从约 150~200ms 降至约 15~25ms，加速约 8~10 倍。

### 5.3 体素降采样尺寸权衡

| voxel_size | 点数减少程度 | 噪声抑制 | 细节损失 | 适用场景 |
|-----------|------------|---------|---------|---------|
| 0.005m | 轻度 | 弱 | 几乎无损 | TSDF 最终输出 |
| 0.01m | 中度 | 中 | 轻微 | ICP 匹配、实时融合 |
| 0.02m | 显著 | 强 | 明显 | 快速预览、粗配准 |

### 5.4 TSDF 体素分辨率权衡

`voxel_length = 0.005m`（5mm）是经验平衡值：

- 更小（如 0.002m）：网格分辨率更高，薄壁结构保留更好，但体素数量呈立方增长，内存和时间开销急剧增大（voxel_length 减半，体素数量增加 8 倍）。
- 更大（如 0.01m）：快速但输出网格粗糙，可能丢失细小特征。

`SDF_TRUNC = 0.04m`（4cm）意味着表面外 4cm 以上的区域不参与 TSDF 更新。这个值需要在截断精度和覆盖范围之间平衡：太小会丢失边界和薄物；太大则增加不必要的体素占用。

### 5.5 滤波模式选择建议

| 模式 | 边缘保持 | 噪声抑制 | 运动适应 | 推荐场景 |
|-----|---------|---------|---------|---------|
| `original` | 差（拖尾） | 好 | 差 | 静态物体、绝对静止场景 |
| `jump` | 好（直接替换） | 好 | 好 | **推荐首选**，一般性使用 |
| `edge` | 最好（主动规避） | 中 | 中 | 富含棱角、孔洞的几何体 |
| `realsense` | 中 | 好 | 中 | 需要稳定 SDK 支持的大型项目 |

`jump` 模式作为推荐首选的原因：它在跳变检测和 EMA 平滑之间取得了良好的平衡——既能保留物体运动或相机移动引起的表面切换信息，又能在静止区域实现有效的噪声抑制。唯一的局限是 `JUMP_THRESHOLD = 0.05m` 的设计假设对于缓慢移动并需要保留亚 5cm 级别变化的场景不够敏感，此时可适当降低阈值。

### 5.6 实时性能概况

在配备 Intel Core i7 / NVIDIA RTX 3060 的工控机上，4 台 D435（1280x720 @15FPS）的运行表现：

| 模式 | 帧率 | 单帧稠密点数 | 备注 |
|-----|------|------------|------|
| 双路实时融合（ICP 每 15 帧） | 8~12 FPS | ~80 万 | 含 Open3D 可视化 |
| 双路实时融合（关闭 ICP） | 12~15 FPS | ~80 万 | 仅外参拼接 |
| TSDF 批量融合（4 cam x 10 帧） | ~5~8 秒/次 | 取决于场景范围 | 离线模式 |

## 总结

本系统的点云融合与滤波子系统围绕"精度-速度-鲁棒性"三角做了系统性的工程设计：

1. **双路点云策略**实现了配准计算开销与输出点云质量的解耦。
2. **Point-to-Plane ICP** 利用工件表面平坦性假设实现稳定收敛，配合 fitness/rmse 阈值判定与累积修正策略，在静态多相机设置中表现出色。
3. **TSDF 体素融合**为离线高质量建模提供水密的点云与网格输出，弥补了实时拼接的重影问题。
4. **四种滤波模式**覆盖了从快速预览到高品质采集的多种场景需求，其中 `jump` 模式作为默认推荐。

整体管线从传感器原始数据到最终可用的几何模型，经历了"空间滤波 → 时序平滑 → 点云生成 → ICP 位姿优化 → 多帧/多视角融合 → 后处理"的完整工序，为下游的轨迹规划提供了可靠的三维几何输入。
